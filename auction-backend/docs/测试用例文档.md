## Phase 1：认证与注册

### TC-1.1 注册卖家用户
- **接口**: `POST /api/user/register`
- **输入**: 用户名、密码（≥6位）、昵称、手机号、邮箱
- **预期**: 返回 200，创建用户成功，返回用户 ID
- **实际**: ✅ 通过

### TC-1.2 重复用户名拦截
- **接口**: `POST /api/user/register`
- **输入**: 已存在的用户名
- **预期**: 返回非 200，拒绝重复注册
- **实际**: ✅ 通过

### TC-1.3 弱密码拦截
- **接口**: `POST /api/user/register`
- **输入**: 密码长度 < 6
- **预期**: 返回非 200，拒绝弱密码
- **实际**: ✅ 通过

### TC-1.4 卖家登录
- **接口**: `POST /api/user/login`
- **输入**: 正确的用户名和密码
- **预期**: 返回 200，返回 JWT Token
- **实际**: ✅ 通过

### TC-1.5 错误密码拒绝

## 安全修复总结

以下安全问题已在测试前修复并验证：

| 编号 | 严重程度 | 问题描述 | 修复方案 |
|------|---------|---------|---------|
| SEC-1 | 🔴 P0 | JWT 伪造：使用 `uid:` 前缀明文传递用户 ID | 使用 jjwt 库实现 HMAC-SHA384 签名 JWT |
| SEC-2 | 🔴 P0 | 权限后门：`userId == 1L` 硬编码管理员权限 | 移除后门代码，强制走 RBAC 鉴权 |
| SEC-3 | 🔴 P0 | 密码明文存储 | 使用 BCrypt 哈希 |
| SEC-4 | 🟠 P1 | WebSocket 跨域通配符 `*` | 限制为 `http://localhost:*` |
| SEC-5 | 🟠 P1 | 前端硬编码后端地址 | 改用环境变量 |
| SEC-6 | 🟠 P1 | 数据库密码硬编码 | 改用环境变量 |
| SEC-7 | 🟡 P2 | 个人信息明文返回（身份证、真实姓名） | VO 脱敏处理 |
| SEC-8 | 🟡 P2 | application.yml 包含敏感配置 | 所有凭证改为环境变量引用 |
